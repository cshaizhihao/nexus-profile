# PRODUCT_AUDIT.md

## 一、结论

Nexus Profile 当前已具备“可运行、可编辑、可部署”的基础能力，但产品完整度仍处于 MVP 早期。主要短板不是功能数量，而是产品叙事、信息架构、后台工作流、开源交付与视觉系统尚未形成闭环。

## 二、目标用户

### 1. 创作者 / 开发者
- 需要一个可自定义的个人主页
- 需要聚合项目、链接、社交身份
- 需要可自己部署，不依赖第三方平台

### 2. VPS / Homelab 用户
- 需要一个轻量导航中心
- 需要 Docker Compose 和一键脚本
- 需要后台管理，不想每次改代码

### 3. 开源用户
- 需要快速安装、快速理解、快速二次开发
- 需要清晰文档、截图、路线图、License

## 三、当前主要问题

### 1. 前台像 Demo，不像产品
- 首页价值主张不够清晰
- Bento 模块偏概念展示，缺少真实个人内容承载
- 缺少项目页、档案页、状态页等个人主页常见模块

### 2. 后台 CMS 还不够产品化
- 页面结构仍偏单页表单
- 缺少独立设置页、主题页、项目管理页
- 缺少危险操作确认、保存状态、表单校验细节

### 3. 开源交付不足
- 缺少 License
- 缺少 CONTRIBUTING / SECURITY
- 缺少截图与部署教程
- 缺少版本路线图

### 4. 设计系统不足
- 颜色、字体、组件状态尚未抽象成稳定 tokens
- 主题预设依赖 CSS 注入，长期应转成配置化 tokens
- 移动端细节需要专项优化

## 四、优先级判断

### Must Have - v0.1.0 必须完成
- 清晰首页产品叙事
- 完整 README
- License
- 后台登录保护
- 基础 CMS CRUD
- Docker Compose 可用
- 一键部署脚本可用

### Should Have - v0.2.0 应完成
- 项目/作品管理
- Profile 档案页
- 数据导入导出
- 主题 tokens 化
- 后台信息架构拆分

### Could Have - v0.3.0 可增强
- 多展示模式
- 公开 API
- 多用户/多站点
- 备份恢复
- 插件化链接卡片

## 五、本轮改造重点

本轮不继续盲目堆功能，优先补齐“产品感”：

1. 首页重构为产品化叙事结构
2. 增加 Profile 和 Projects 路由
3. 增加开源交付文件
4. 补 README 使用与部署说明
5. 建立 ROADMAP
